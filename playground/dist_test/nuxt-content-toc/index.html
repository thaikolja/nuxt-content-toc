A highly customizable and accessible Table of Contents component for @nuxt/content v2 and v3. Provides automatic heading extraction, scroll synchronization, and active section highlighting.

Installation

Install the module using your preferred package manager:

# npm
npm install nuxt-content-toc

# yarn
yarn add nuxt-content-toc

# pnpm
pnpm add nuxt-content-toc

Add it to your nuxt.config.ts:

export default defineNuxtConfig({
  modules: ["nuxt-content-toc", "@nuxt/content"],
});

That's it! The <TableOfContents /> component is now globally available.

Basic Usage

The simplest way to use the component:

<template>
  <TableOfContents />
</template>

This will automatically extract headings from the current page's content.

Passing Content Directly

To avoid double-fetching, pass the content object directly:

<script setup>
const { data } = await useAsyncData("page", () =>
  queryCollection("content").path("/docs").first(),
);
</script>

<template>
  <TableOfContents :page="data" />
</template>

Configuration

Configure the module globally in nuxt.config.ts:

export default defineNuxtConfig({
  modules: ["nuxt-content-toc"],

  contentToc: {
    // Heading levels to include (h2, h3, h4)
    levels: [2, 3, 4],

    // CSS class for active TOC items
    activeClass: "toc-active",

    // CSS class for each TOC item
    itemClass: "toc-item",

    // CSS class for nested sublists
    sublistClass: "toc-sublist",

    // CSS class for sublist items
    sublistItemClass: "toc-sublist-item",

    // Offset from viewport top (for fixed headers)
    scrollOffset: 80,
  },
});

Component Props

Override global settings per-component:

PropTypeDefaultDescription
levelsnumber[][2, 3, 4]Heading levels to include
activeClassstring'toc-active'Class for active items
itemClassstring'toc-item'Class for each item
sublistClassstring'toc-sublist'Class for nested sublists
sublistItemClassstring'toc-sublist-item'Class for sublist items
scrollOffsetnumber80Top offset in pixels
pathstringCurrent routeCustom content path
pageobjectundefinedContent object (avoids double-fetch)
ariaLabelstring'Table of Contents'ARIA label for navigation

Composables

useContentToc

Extract TOC data programmatically:

const { toc, headingIds, pending, error, refresh } = useContentToc();

Returns:

  • toc — Reactive array of TocItem objects
  • headingIds — Flat array of all heading IDs
  • pending — Loading state
  • error — Error state if fetch fails
  • refresh() — Re-fetch TOC data

With content object:

const { data } = await useAsyncData("page", () => queryContent("/").findOne());
const { toc } = useContentToc(data);

useActiveHeading

Track which heading is currently in view:

const { headingIds } = useContentToc();
const activeId = useActiveHeading(headingIds);

Uses IntersectionObserver for optimal performance.

useContentVersion

Detect the installed @nuxt/content version:

const version = useContentVersion(); // 'v2' or 'v3'

Styling

CSS Custom Properties

Customize the appearance using CSS variables:

:root {
  /* Typography */
  --toc-font-size: 0.875rem;
  --toc-line-height: 1.5;

  /* Colors */
  --toc-text-color: #374151;
  --toc-text-color-hover: #111827;
  --toc-active-color: #3b82f6;

  /* Borders */
  --toc-border-color: #e5e7eb;
  --toc-active-border-color: #3b82f6;

  /* Spacing */
  --toc-indent: 1rem;

  /* Animation */
  --toc-transition-duration: 0.15s;
}

Dark Mode

Variables are automatically adjusted for dark mode via prefers-color-scheme:

@media (prefers-color-scheme: dark) {
  :root {
    --toc-text-color: #d1d5db;
    --toc-text-color-hover: #f9fafb;
    --toc-active-color: #60a5fa;
    --toc-border-color: #374151;
    --toc-active-border-color: #60a5fa;
  }
}

Custom Styling Examples

Minimal Style:

.toc-link {
  color: inherit;
  text-decoration: none;
}

.toc-link.toc-active {
  color: #3b82f6;
  font-weight: 600;
}

Left Border Indicator:

.toc-link {
  border-left: 2px solid transparent;
  padding-left: 0.75rem;
  transition: all 0.15s ease;
}

.toc-link.toc-active {
  border-left-color: #3b82f6;
  color: #3b82f6;
}

Nested Highlighting:

/* Highlight parent when child is active */
.toc-item:has(.toc-active) > .toc-link {
  color: #60a5fa;
  border-left-color: #60a5fa;
}

Accessibility

The component follows WAI-ARIA guidelines:

ARIA Attributes

  • role="navigation" — Identifies as navigation landmark
  • aria-label — Provides context for screen readers
  • aria-current="location" — Marks the active section

Keyboard Navigation

  • Tab — Navigate between TOC items
  • Enter / Space — Activate the focused item
  • Smooth scroll to target heading

Screen Reader Support

  • Loading state announces "Loading table of contents..."
  • Error state includes role="alert"
  • Hidden .sr-only announcements for state changes

TypeScript

Full TypeScript support with exported types:

import type {
  TocItem,
  UseContentTocReturn,
  UseActiveHeadingOptions,
  ContentTocRuntimeConfig,
} from "nuxt-content-toc";

TocItem Interface

interface TocItem {
  id: string; // Heading anchor ID
  depth: number; // Heading level (2, 3, 4...)
  text: string; // Heading text content
  children?: TocItem[]; // Nested headings
}

Version Compatibility

nuxt-content-tocNuxt@nuxt/content
1.x3.x / 4.x2.x / 3.x

The module automatically detects the content version and uses the appropriate API:

  • v3: queryCollection().path().first()
  • v2: queryContent().findOne()

Runtime Access

Access module config and version at runtime:

const { $contentToc } = useNuxtApp();

console.log($contentToc.version); // '1.0.0'
console.log($contentToc.config.activeClass); // 'toc-active'

Examples

Documentation Sidebar

<template>
  <aside class="docs-sidebar">
    <h2>On This Page</h2>
    <TableOfContents :levels="[2, 3]" :scroll-offset="100" />
  </aside>
</template>

Blog Post TOC

<script setup>
const { data: post } = await useAsyncData("post", () =>
  queryCollection("content").path(route.path).first(),
);
</script>

<template>
  <article>
    <ContentRenderer :value="post" />
  </article>

  <aside>
    <TableOfContents :page="post" />
  </aside>
</template>

Custom Active Indicator

<script setup>
const { toc, headingIds } = useContentToc();
const activeId = useActiveHeading(headingIds);
</script>

<template>
  <nav>
    <a
      v-for="item in toc"
      :key="item.id"
      :href="`#${item.id}`"
      :class="{ active: activeId === item.id }"
    >
      {{ item.text }}
    </a>
  </nav>
</template>
